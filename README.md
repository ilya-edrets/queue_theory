# Основы теории массового обслуживания для Highload-систем

Учебный курс о природе задержек (latency) и методах их анализа в современных
IT-системах — на примерах ML-инференса и Kafka.

## Структура курса

**Модуль 1. Базовые законы и природа задержек**
1. [Из чего состоит Latency?](lessons/01-latency-components.md) — transmission vs propagation time
2. [Теорема Литтла — «закон сохранения» в очередях](lessons/02-littles-law.md) — $L = \lambda W$

**Модуль 2. Моделирование и распределения**
3. [Нотация Кендалла и классификация систем](lessons/03-kendall-notation.md) — A/S/k, типы M/D/G
4. [Свойство «памяти» и тяжёлые хвосты](lessons/04-memoryless-heavy-tails.md) — экспонента, Парето, failure rate

**Модуль 3. Практические эффекты и масштабирование**
5. [Эффект утилизации и формула Поллачека — Хинчина](lessons/05-utilization-pollaczek-khinchine.md) — почему нельзя грузить систему на 100%
6. [Микробёрсты и парадокс инспекции](lessons/06-microbursts-inspection-paradox.md) — чего не видит мониторинг
7. [Масштабирование и планировщики](lessons/07-scaling-schedulers.md) — square-root staffing, FIFO vs Processor Sharing

**Итоговая работа**
8. [Разгадка SelfPing](lessons/08-final-selfping.md) — почему пустые задачи
   каждые 10 мс видят то, чего не видят графики утилизации; финальный квиз
   по всему курсу

## Структура репозитория

```
study_plan.md          # исходный план курса
lessons/               # уроки в Markdown
  NN-<slug>.md         # текст урока: теория, графики, квизы, задачи
  STYLE_GUIDE.md       # стайлгайд для авторов уроков (не является уроком)
  images/              # PNG-графики к урокам
  scripts/             # Python-скрипты генерации графиков (matplotlib)
docs/                  # готовый статический сайт курса (генерируется)
  index.html           # обложка курса с прогрессом
  lesson-NN.html       # страницы уроков с интерактивными квизами
  assets/              # стили и JS (квизы, прогресс в localStorage)
  images/              # копия графиков
tools/
  build_site.py        # сборка сайта из markdown-уроков
```

## Формат урока

Каждый урок содержит:
- **TL;DR** — суть в 2–4 предложениях;
- объяснение «от интуиции к формуле» с аналогиями из жизни;
- 2–4 графика и сквозные примеры (ML-инференс на GPU, Kafka);
- **Проверь себя** — 4–6 вопросов-квизов с пояснениями;
- **Задачи** — расчётные задачи с решениями под спойлером.

## Перегенерация графиков

Графики строятся скриптами из `lessons/scripts/`:

```bash
python3 -m venv .venv
.venv/bin/pip install matplotlib numpy
.venv/bin/python lessons/scripts/lesson01_charts.py   # и т.д. для каждого урока
```

Запускать из корня проекта — пути к `lessons/images/` относительные.

## Сайт

Готовый статический сайт лежит в `docs/` — без бэкенда и авторизации,
открывается на ПК и телефоне. Достаточно открыть `docs/index.html` в браузере
(двойным кликом, без сервера). Интерактив на ванильном JS: квизы с мгновенной
проверкой и пояснениями, прогресс прохождения в `localStorage`, формулы
рендерит KaTeX (подгружается с CDN — для формул нужен интернет).

После правок в `lessons/*.md` сайт пересобирается одной командой:

```bash
.venv/bin/pip install markdown   # один раз
.venv/bin/python tools/build_site.py
```

приводятся только там, где они дают понимание.
