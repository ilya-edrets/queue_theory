# Добавление / переработка урока

## Перед началом

1. Прочитай `lessons/STYLE_GUIDE.md` — формат урока обязателен (структура,
   строгий формат квизов, шаблон matplotlib-скриптов, сквозные примеры
   ML-инференс/Kafka, тон, «ё»).
2. Прочитай `study_plan.md` — охват тем и целевую аудиторию.
3. Посмотри соседний готовый урок (например `lessons/05-*.md`) как эталон.

## Шаги

1. **Текст**: `lessons/NN-<kebab-slug>.md`. Для нового урока NN — следующий
   свободный двузначный номер; добавь его в список `MODULES`
   в `tools/build_site.py` (иначе урок не попадёт на обложку сайта) и в
   оглавление `README.md`.
2. **Графики**: `lessons/scripts/lessonNN_charts.py` по шаблону из стайлгайда,
   PNG → `lessons/images/lessonNN_<name>.png`. Запуск из корня:
   `.venv/bin/python lessons/scripts/lessonNN_charts.py`. Все подписи русские.
   Случайность — только с фиксированным seed (воспроизводимость).
3. **Сверь числа**: каждое число в тексте урока (примеры, ответы задач) должно
   сходиться с расчётом — проверь выкладки скриптом или вручную.
4. **Валидация**: `.venv/bin/python tools/validate_lessons.py`.
5. **Сайт**: `.venv/bin/python tools/build_site.py`, затем
   `.venv/bin/python tools/validate_lessons.py --site`.
6. **Визуально** (по необходимости): скриншот страницы headless-Chrome —
   команда в CLAUDE.md.

## Частые ошибки

- Несколько `- [x]` или ни одного в вопросе — сайт сломается; ровно один.
- Пояснение к вопросу обязано начинаться с `> **Пояснение:**`.
- Служебные пометки автора («на реальном сайте…», TODO) в тексте урока —
  валидатор их ловит, но лучше не оставлять.
- Правка `docs/lesson-NN.html` руками — изменения сотрутся при пересборке;
  источник истины — markdown.
- В формулах десятичная запятая записывается как `0{,}8`, иначе KaTeX
  отрендерит лишний пробел.
